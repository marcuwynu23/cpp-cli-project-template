# Documentation

## [Command-Line Guide](Command-Line-Guide.md)

Complete guide to building a **professional command-line interface** with CLI11 in this project:

- **Options** — named arguments (`--file`, `-f`), types, defaults, required, multiple values
- **Flags** — boolean switches, counting flags (`-vvv`)
- **Positional arguments** — ordered arguments, vectors, required
- **Subcommands** — `app build`, `app run`, chained subcommands
- **Validation** — ranges, enums, existing file/path, custom validators
- **Help and version** — `--help`, `--version`, descriptions, footer
- **Option groups** — grouping in help
- **Error handling and exit codes**
- **Professional CLI design checklist**
- **Full example** — multi-subcommand app with global options

Use this guide when adding or extending CLI options and subcommands in the app (e.g. in `Controller`).
